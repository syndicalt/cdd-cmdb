# CDD-CMDB

**A Configuration Management Database defined entirely as a test suite.**

This project does not contain a CMDB. It contains the *specification* for one: a comprehensive set of tests that define the required behaviors, interfaces, data models, and invariants. Any implementation that passes the tests is a valid CMDB instance — whether it's generated by AI, hand-written, or assembled from existing services.

## Why

Traditional CMDB products are monolithic, hard to customize, and create vendor lock-in. This project inverts the model:

- **Ship tests, not code.** The test suite is the product. Implementations are disposable.
- **Generate on demand.** The included generator uses an LLM to produce a passing implementation from scratch, targeting any backend stack.
- **Upgrade by re-testing.** Pull new test suites and regenerate. No migrations — new instances inherently comply.

## Quickstart

```bash
git clone https://github.com/syndicalt/cdd-cmdb.git && cd cdd-cmdb
./demo.sh
```

Or with Docker:

```bash
docker compose up --build
```

## Test Suites

| Suite | What it specifies |
|---|---|
| `core/` | CI CRUD, relationships, schema validation |
| `search/` | Full-text search, attribute filtering, wildcards, sorting, pagination |
| `diff/` | Attribute-level change diffs, point-in-time snapshots |
| `reconciliation/` | Source-based reconciliation (new/updated/unchanged/stale) |
| `tags/` | Tag CRUD, tag-based search, tag listing |
| `ttl/` | Per-CI expiry, sweep lifecycle, status filtering |
| `webhooks/` | Webhook subscriptions, test pings, delivery history |
| `security/` | Injection resistance, auth, RBAC |
| `discovery/` | Bulk CI import, source metadata |
| `performance/` | Latency SLAs, throughput under load |
| `governance/` | Runtime validation policies |
| `audit/` | Immutable CI change history |
| `graph/` | Multi-hop impact analysis, dependency traversal |

## Profiles

| Profile | Suites | Use case |
|---|---|---|
| `minimal` | core | MVP — prove basic CRUD works |
| `standard` | core, discovery, audit, graph, search, diff, reconciliation, tags, ttl, webhooks | Production-grade with full data model |
| `enterprise` | all | Full compliance including security, performance, governance |
